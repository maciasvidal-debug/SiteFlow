## 2024-05-24 - Avoid Date Object Instantiation in Loops
**Learning:** Instantiating `new Date()` objects inside a loop for simple date comparisons is a significant performance bottleneck in Javascript. When dealing with ISO 8601 formatted date strings (YYYY-MM-DD), Javascript can perform string comparison efficiently, rendering object instantiation unnecessary and slow.
**Action:** When filtering or comparing dates within a loop, calculate the bounds outside the loop as ISO 8601 strings and perform direct string comparisons (`>=`, `<=`) on the raw string data instead of converting each iteration's value into a Date object.
